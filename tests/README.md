# Tests Directory

This directory contains unit tests, integration tests, and validation scripts.

## Guidelines:
- Write comprehensive tests for all critical functionality
- Include both unit tests and integration tests
- Use consistent testing frameworks and conventions
- Maintain test data and fixtures as needed

## File Organization:
- Mirror the structure of the source code directory
- Use descriptive test function names
- Include test data in appropriate subdirectories
- Document test requirements and setup procedures
